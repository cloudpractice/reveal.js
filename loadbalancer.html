Now that we have created a web site using a single web server, we will extend the solution to include a second web server and a load balancer.

When web surfers attempt to access our web site, the load balancer will spread the requests across our web instances.

Create a Load Balancer.

To add a load balancer, select the Load Balancers link in the Network navigation pane.

Add a Load Balancer Pool.

The main view lists the Load Balancer Pools for this project. At present, the list should be empty.

To create a new Load Balancer Pool, select + Add Pool. 

NOTE: The Pool is a container that holds instances

A pop-up window will appear.

Add a Load Balancer Pool. Step One.

In the Name field, type a name for your pool. A good choice is web-pool

In the Subnet drop-down menu, select the private network.

In the Protocol drop-down menu, select HTTP

In the Load Balancing Method drop-down menu, select ROUND_ROBIN

All other fields can be left to their default values.

Once completed, select Add to add your Load Balancer Pool.

The private network should be on the only choice in the drop-down list.

Load Balancer Pool Created.

Once completed, a new pool entry will appear in the Pools list.

The next step is to add a VIP to the pool. To do this, select More then Add VIP.

NOTE: The VIP is the IP address that users will type into their web browser to reach your web site.

A pop-up window will appear.

The name should be web-pool

Add a VIP. Step One.

In the Name field, type a name for your VIP. A good choice is web-vip.

In the Specify a free IP address ... field, type a free IP address. A good choice is 192.168.55.35.

In the Protocol Port field, type 80

In the Session Persistence drop-down menu, select SOURCE_IP

Once completed, select Add to add your VIP. You will be returned to the Load Balancer view.

In the Protocol drop-down menu, select HTTP

VIP Added.

Once completed, the VIP name (web-vip) will appear in the web-pool Pools entry.

The next step is to add instance members to the pool. To do this, select the Members tab.

Add First Pool Member

The main view lists the instance members in the Load Balancer Pool.

To add a new instance Member to the Load Balancer pool, select + Add Member.

At present, the list should be empty.

A pop-up window will appear.

Add First Pool Member. Step One.

In the Pool drop-down menu, select web-pool.

In the Members selection, tick web-app-instance.

In the Protocol Port field, type 80.

Once completed, select Add to add the member instance.

All other fields can be left to their default values.

First Pool Member Added.

Once completed, the new member instance will be added to the web-pool Pool.

The next step is to add a public IP address to the Pool. To do this, select the Access & Security link in 

the Compute Navigation Pane

Add a Public IP Address. Step One

If not already selected, select the Floating IPs tab to display the list of available floating IPs.

To allocate another Floating IP, select the Allocate IP To Project button.

Public IP Address Added

On completion, a second Public IP address will be allocated to the project.

To associate the Public IP address with the VIP, select Associate.

A pop-up window will appear.

Associate Public IP Address

In the Port to be associated drop-down menu, select the None: 192.168.55.35.

To complete the association, select the Associate button.

You will be returned to the Floating IPs view.

This will associate the Public IP address to the VIP.

Public IP Address Associated.

On completion, the IP address will be associated with the VIP.

The next step is to add a second web server instance. To start, select Instances from the Navigation pane.

NOTE: In the Instance column, the value should be “-“.

Challenge Time!

For this second web server, we have not provided any instructions, but you've done it before.

Add Second Instance to Pool.

On completion, a second web server instance will appear in the Instances list.

The next step is to add this second web server instance to the load balancer. Select Load Balancers link 

in the Network Navigation pane.

Add Second Instance to Pool. Step One.

If it is not already selected, select the Members tab.

The existing web server instance should be in the Member list.

To add the second web server instance, select + Add Member.

A pop-up window will appear.

Second Instance Added to Pool.

On completion, there should be two instance members in the Members list.
